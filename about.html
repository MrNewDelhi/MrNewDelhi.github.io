How I work

  • I default to building tools over heroics. If the same toil shows up twice, I write the thing that kills it.
  • I write things down. The 1,658-line test case writing guide that powers the AIO TCC skill started as a Notion doc.
  • I sit in the bug triage. I sit in the PM standup. QA is a cross-team function and the lift comes from sitting in the right rooms.
  • I prefer deterministic over clever. A boring suite that runs the same way every time beats a magical one that fails on Tuesdays.
